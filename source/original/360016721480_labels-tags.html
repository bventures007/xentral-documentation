Tags markieren wichtige Vorgänge oder Belege in Übersichten wie z.B. zu prüfende Aufträge in der Auftragsübersicht. Die Tags können beliebig vergeben und in Tabellen gesetzt werden. Beispielsweise können zu priorisierende Aufträge markiert werden. Ebenfalls Klärfälle aus der Logistik, die der Kundenservice nochmals zur Prüfung erhält oder im B2B Bereich, falls eine Adressprüfung im Account Management noch nicht vollständig erledigt ist usw.

Tags können als Markierung verschiedenster Vorgänge verwendet werden:

  • Prio-Markierung: Prio Stufen und Farben

  • Prüf-Markierung: Übergabe an andere Teams / Markierung für andere Team

  • Status & Information: Informationen zum Status spezieller Fälle

Tag-Typen

Unter Tags > Tag- Typen findest du eine Auflistung aller angelegten Tags.

Neues Tag

Neue Tags können über den Button +NEU  angelegt werden.

Die Daten sind in die Felder einzutragen und zu speichern:

  • Bezeichnung:, mit der das Tag in der Oberfläche angezeigt wirdinter

  • Kennung: Systeminterne Kennung des Tags (ohne Leerzeichen und Sonderzeichen)

  • Gruppe: Teilt das Tag einer Gruppe zu, wie z.B Adresse, Gutschrift oder Retoure

Ein neues Tag kann auch direkt beim Markieren eines Eintrages mit einem Tag angelegt werden. Hierzu ist auf das Tag-Icon  zu klicken und der Button in der Tag-Übersicht auszuwählen.

Tags verwenden

Tags können in folgenden Modulen verwendet werden:

  • Stammdaten: Adressen, Artikel, Projekte

  • Verkauf: Anfrage, Angebote, Aufträge, Gutschriften, Lieferscheine

  • Einkauf: Preisanfrage, Bestellungen, Produktion

Ein Tag wird direkt über die Listenansicht vergeben. Klicke dazu auf das Tag- Symbol rechts. Für diesen Eintrag können ein oder mehrere Tags als Markierung vergeben werden. Die Tags werden direkt in der Liste angezeigt. Es besteht auch die Möglichkeit neue Tags zu erstellen oder zu bearbeiten.

Automatische Tags verwenden

Ab Version 19.4 gibt es zusätzlich die Möglichkeit automatische Tags festzulegen die nach dem Drucken des Produktions-Belegs gesetzt werden. Zunächst wird das Tag selbst angelegt und anschließend wird im Tab Automatische Tags definiert, welches Tag für welchen Beleg automatisch gesetzt werden soll, sobald der Beleg gedruckt wurde. In folgenden Beispiel wird das Tag "gedruckt" automatisch gesetzt sobald die Produktion ausgedruckt wird.

  • Lege wie zuvor beschrieben ein Tag an

  1. Klicke im Tab Automatische Tags auf +Neuer Eintrag

  2. Wähle dein Tag und wann es gesetzt werden soll (Drucken von Beleg oder Autoversand)

  3. Wähle in welchem Gebiet dein Tag Anwendung findet (z.B. Angebot, Gutschrift oder Produktion)

In der Übersicht des Tabs Automatische Tags sind alle festgelegten Regeln zu sehen.

In dem zuvor aufgezeigten Beispiel wird das Tag "gedruckt" automatisch gesetzt, sobald in der Produktionsübersicht eine Produktion ausgewählt und über die Stapelverarbeitung gedruckt wird.

Übertragen Modul verwenden

Im Modul Übertragungen besteht die Möglichkeit, dass auch automatisch durch den Eingang von Daten Tags erstellt werden. Dies ist aber nur der Fall, wenn es zur Anlage von Belegen kommt.

Die Funktion ist aktuell auf folgende Belegtypen abgestimmt:

'angebot' , 'auftrag' , 'bestellung', 'gutschrift', 'lieferschein', 'produktion', 'rechnung'

Praxisbeispiele für Tags

Prio-Markierung

Tags können als Markierung für die Priorisierung und die schnelle Sicht in der Auftragsübersicht verwendet werden. Das Tag steuert hier nicht selbständig in die Logistik ein, hierfür gibt es zusätzlich die Priorisierung über die Fast-Lane Markierung (kann über den Shop oder manuell im Auftrag gesetzt werden und priorisiert im Logistikprozess oder Fulfillment die Versandaufträge).

Prüf-Markierung

Über das Tag kann eine Prüfmarkierung für andere Teams gesetzt werden z.B. bei Kundenrückfragen zu Aufträgen. Das Tag blockiert die Aufträge nicht für die Logistik. Hierfür kann im Auftrag der Haken für den Autoversand erlauben oder der Liefertermin verwendet werden.

Statusinformation

Statusinformation für Adressen beispielsweise die Prüfung einer Adresse für den B2B Bereich (Umsatzsteuer ID Prüfung für EU, Export Prüfung, Adressprüfung von Neukunden (Großhandelskunden)).

Info-Tag für Sortierung

Informations-Tags, die nicht abgearbeitet, sondern für die Sortierung und Klassifizierung gesetzt werden.

Sortierung der Labels in der Auswahlliste

Die Labels werden nach Erstellungs-ID sortiert. Um Labels umzusortieren kannst du die Labels überschreiben.
