## Gutschriften

- Gutschriften werden von der Amazon Schnittstelle immer in xentral erzeugt, auch wenn der Kunde den VAT Service nicht nutzt
- Gutschriften basieren auf der dazugehörigen Rechnung (bzgl. Positionen und Wert)
- In der Regel gibt es eine Gutschrift pro Auftrag über die retournierten Artikel, ggf. kann es zu mehreren Gutschriften kommen, falls der Kunde einzelne Positionen separat retourniert (08.02, AB, Infos von Christian DIGITALXL)

## Amazon VAT Services: Rechnungserstellung

Bei Nutzung des Amazon VAT Services wird ein anderer Nummernkreis bei Rechnungen verwendet (INV-...-...). (AB, 30.12.20)

Es ist normal, dass es zu einem FBA Auftrag mehrere Rechnungen gibt, die über den VAT Service importiert werden. Dies geschieht v.a. wenn der Auftrag bei Amazon von verschiedenen Amazon Lägern bedient wird. Pro Lager gibt es dann idR eine Rechnung. (AB, 08.02.2022, Quelle: DigitalXL)

Bei Nichtnutzung von VAT: Bei PanEU und Mitteleuropa Programm kann eine korrekte Rechnungserstellung durch xentral nicht gewährleistet werden, weil xentral nicht auswertet von welchem Lager die Ware verschickt wird, was wiederum die Grundlage für die korrekte Versteuerung ist. Kunde benötigt in diesem Fall amainvoice. (AB, 08.02.2022, Quelle: DigitalXL)

## Erstattungen FBM

- Gutschriften können in Xentral erstellt werden und werden - wie bei PayPal Rückerstattungen - über den Zahlungsverkehr erstattet
- Als Geschäftskonto wählt man z.B. seine eigene Bank, da bei FBM eine Erstattung von Xentral über Amazon nicht möglich ist
- Die in Xentral erstellte manuelle Erstattung wird nicht an Amazon gemeldet
- Amazon kann auch selbst die Erstattung auslösen, wenn sich der Kunde bei FBM nicht an den Händler, sondern direkt an Amazon wendet. Diese Erstattungen von Amazon werden über den V2 Bericht importiert. => hier aufpassen, dass nicht doppelt erstattet wird (1x von Amazon und 1x vom Händler selbst) (AB, 09.04.21)

## Amazon Pay

- Zahlungsweise Shopsystem
- Zahlungseingang nur über CSV
- Ausgang (Gutschriften / Erstattungen) via API (AB, 09.04.21)

## Amazon Payments / Zahlungsberichte / V2 Bericht

- Der V2 Bericht wird über den Amazon Importer importiert - ist im Finanzreport zu finden
- Für die Verbuchung ist das Geschäftskonto Amazon Zahlungsberichte notwendig
- Der V2 Bericht muss zuvor bei Amazon freigeschaltet werden (AB, 09.04.21)
- Sollte ein Zahlungsbericht fehlen/ ausgelassen worden sein (wegen Bug oder ähnlichem, siehe FAC-431 z.B.), dann kann dieser rückwirkend manuell in xentral hochgeladen werden. Hake dazu in der Shopschnittstelle von Amazon unter Finanzreport die betroffene/n Datei/en an und setze diese über die Stapelverarbeitung "auf nicht importiert"/ "importiert nein". Gehe dann in das Modul Zahlungseingang und starte den Live-Import erneut.

## Zahlungsweise amazon ohne Geschäftskonto (16.08.2021, AB)

- FBA/FBM: Zahlungen werden von Schnittstelle erzeugt und nicht via Autoversand
- Problem: Zahlungen werden nicht als bezahlt markiert, obwohl dies in der Zahlungsweise eingestellt ist
- Grund: Mahnwesen + Prozessstarter muss für dieses Projekt aktiviert sein, weil hierdurch die Zahlung manuell durch den Prozessstarter auf bezahlt gesetzt wir

## Amazon Seller App

- Wird eine Anlieferung für eine JIT Stückliste an Amazon erstellt, die über Amazon FBA verkauft werden soll, ändert sich der "Lagerbestand bei Amazon" nicht sondern bleibt immer auf 0. Diese Funktion funktioniert nur für Lagerartikel.
