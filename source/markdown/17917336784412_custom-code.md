## Wichtige Informationen für die Nutzung von Custom Code in deiner Xentral-Instanz

Wenn du darüber nachdenkst, Custom Code in deiner Xentral-Instanz einzusetzen, oder bereits bestehenden Custom Code nutzt, möchten wir dir die wichtigsten Aspekte mit auf den Weg geben. Unser Ziel ist es, dich dabei zu unterstützen, die beste Lösung für dein Unternehmen zu finden – effizient, sicher und zukunftsfähig.

## Warum Custom Code eine bewusste Entscheidung sein sollte

Custom Code ermöglicht es, spezifische Anforderungen umzusetzen, die über den Standard von Xentral hinausgehen. Das kann in manchen Fällen sinnvoll sein, bringt jedoch auch Herausforderungen mit sich. Custom Code erhöht die Komplexität deines Systems und birgt Risiken, etwa dass er nach einem Update nicht mehr funktioniert. Gründe hierfür können unter anderem Änderungen an **Datenstrukturen **, ** Funktionen **, der ** Datenbankversion **oder der ** PHP-Version**sein.

Stelle dem Nutzen, den dir eine kundenspezifische Funktionalität bringen soll, immer die Kosten der Erstellung, der Wartung und des Betriebs gegenüber, um zu entscheiden, ob das Investment langfristig für dich rentabel ist. Prozesse, die über unsere API gebaut werden, sind in der Regel sinnvoller und langfristig kostengünstiger.

**Deshalb empfehlen wir dir, Prozesse, wann immer möglich, über unsere API zu lösen.**

## Unsere Empfehlung: Setze auf die API

Die[Xentral API](https://help.xentral.com/hc/de/articles/6618896113820#UUID-03a6d5d2-0c6b-65c8-6753-4bc5ad5bb509)ist leistungsstark, flexibel und wird kontinuierlich weiterentwickelt. Mit der API kannst du Prozesse automatisieren, Systeme nahtlos verbinden und maßgeschneiderte Lösungen entwickeln – ohne die Nachteile von Custom Code. Solltest du spezielle Anforderungen haben, die derzeit nicht über die API abgedeckt werden, lass es uns wissen. Unser Team prüft gerne, ob weitere Endpunkte ergänzt werden können. ## Bestehenden Custom Code weiter nutzen oder entfernen?

Wenn du bereits Custom Code nutzt, stehen dir zwei Optionen offen:

1. **Beibehaltung des Custom Codes:** Du kannst deinen bestehenden Code weiter nutzen, solltest jedoch bedenken, dass die Verantwortung für dessen Wartung und Funktionalität bei dir liegt. Sollte dein Custom Code durch ein Update beeinträchtigt werden, können Kosten für Anpassungen entstehen.
1. **Entfernung des Custom Codes:** Wenn der Custom Code nicht mehr benötigt wird oder durch eine API-Lösung ersetzt werden kann, helfen wir dir gerne bei der Deaktivierung oder Umstellung.

## So gehen wir mit neuen Custom Code-Anfragen um

Wenn du neuen Custom Code einführen möchtest, prüfen wir zunächst, ob dieser den Richtlinien entspricht und ob er nicht durch eine API-Lösung abgedeckt werden kann. Sollte der Custom Code zugelassen werden, unterstützen wir bei der Integration. Bitte sende uns deine Anfrage mit dem Betreff **Custom Code** an[support@xentral.com](https://help.xentral.com/hc/de/requests/new), und unser Team wird dich durch den Prozess begleiten.

Bitte beachte, dass bei der Nutzung von Custom Code ein zusätzlicher Serviceaufwand entstehen kann, insbesondere wenn dein spezifischer Anwendungsfall nicht außerhalb des Systems oder über eine API-Lösung abgedeckt werden kann. Für den regelmäßigen Betrieb und die Verwaltung können monatliche Kosten anfallen, die unter anderem Uploads und technische Prüfungen abdecken. Beachte jedoch, dass die Stabilität und Sicherheit von Custom Code ausschließlich in deiner Verantwortung liegen und Xentral hierfür keine Garantien übernehmen kann.

## Wie wir dich unterstützen

- **Prüfung:** Bei neuen Anfragen analysieren wir den zugrunde liegenden Use-Case und helfen dir zu verstehen, ob der Einsatz von Custom Code noch notwendig ist oder durch eine modernere API basierte Lösung ersetzt werden kann.
- **Klarheit über Zuständigkeiten:** Wir übernehmen die technischen Prüfungen, während du oder dein Partner sich um die Erstellung und Anpassung des Codes kümmern.
- **Unterstützung bei der Migration:** Falls du deinen Custom Code durch die API ersetzen möchtest, stehen wir dir beratend zur Seite. Je nach Umfang der Migration können hier zusätzliche Kosten entstehen.

## Warum jetzt handeln?

Die Nutzung von Custom Code ist eine bewusste Entscheidung, die langfristige Auswirkungen auf die Stabilität und Wartbarkeit deines Systems hat. Mit unserer API-Infrastruktur kannst du nicht nur Kosten sparen, sondern auch sicherstellen, dass dein System stets auf dem neuesten Stand bleibt. Ein weiterer Vorteil unserer API ist, dass sie durch automatische Tests abgedeckt ist, wodurch gewährleistet wird, dass sie auch nach Updates weiterhin einwandfrei funktioniert – im Gegensatz zu Custom Code, der bei einem Update evtl. angepasst werden muss.

Wenn du Fragen hast oder Unterstützung benötigst, melde dich gerne bei uns. Gemeinsam finden wir die beste Lösung für dein Unternehmen und machen deine Xentral-Instanz fit für die Zukunft.
